What Was Measured

The sweep used 28 shapes.

Base shapes use K in 4096, 8192 and N in 4096, 8192, 11008.

Extra down-proj shapes use K=11008 and N=4096.

The reference computes x.float() @ dequant(wq).T, then casts to bf16.

TF32 was disabled. CUDA events measured benchmark latency.

NCU counters required sudo -n ncu on this instance.

Comparable Kernels

Library or kernel Research finding H100 status Decision
MARLIN Closest FP16xINT4 LLM kernel. Targets group size 128. Clone worked. Package was not installed. No number. Packing differs and README excludes Hopper tuning.
vLLM AWQ-Marlin Uses packed int32 qweight, qzeros, and scales. Not installed. No number. Requires model executor integration.
TensorRT-LLM Documents W4A16 AWQ/GPTQ support on Hopper. Not installed. No number. Requires engine build and model format.
TensorRT Edge-LLM GEMV Documents INT4 groupwise GEMV for M=1..4. Not installed. No number. Contract differs from this repo.
bitsandbytes Linear4bit Implements FP4/NF4 blockwise layers. Not installed. No number. Quantization is not equivalent.
CUTLASS Supports int4 types and GEMM building blocks. Not installed as Python package. No number. It is not a drop-in W4A16 layer.

Idea Result Evidence

Next Useful Work

Do not claim full model latency yet.

The repo still lacks real checkpoint quantization.

Do not keep the tested symmetric-zero path.

Try a swizzled weight layout. It could improve coalescing across N tiles.

Try M-specific N_TILE autotuning. The large M=8 case is memory-heavy.

Integrate one real packing format before comparing against Marlin or vLLM.
